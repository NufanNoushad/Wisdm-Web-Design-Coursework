Home Page

My name is Nufan and my role is Student 2, I designed and developed the home page, global template & CSS, table page.

Purpose: The home page is the main entry point where users will land when the enter the website. It serves as the main hub and it allows you to navigate to other pages of the website.

The Quality Education's home page uses a structured combination of HTML and CSS to create a visually appealing, functional and a responsive page. Below I have explained the key HTML elements and CSS used to build this page, and also explaining their logic and purpose.

Home Page's HTML / CSS Structure and Key Features

  • The DOCTYPE HTML tag makes sure that the document is written in HTML5
  • The html lang en tag ensured that the document is written in English for accessibility.
  • The header section includes the website logo for branding, which is clickable and returns users to the home page.
  • I used the height technique using CSS in order to style the logo size consistently across each page
  • For the navigation bar, I utilised CSS hover effects so that when the user hovers it highlights active links
  • I implemented flexbox for better alignment and used W3Schools Flexbox Syntax
  • For the mission statement section, I made use of h2 and p tags in order to communicate the website's goal
    and purpose and applied the CSS technique of justify content for spacing and readability.
  • Footer section: Includes a direction link to this Page Editor, uses a fixed position (CSS) to stay visible even when scrolling up and down.

Home Page's Accessibility

To ensure the home page was accessible, I made sure to follow accessibility principles such as
semantic HTML. I used appropriate alt text to all images, ensuring that users with visual
impairment would be able to understand the content. Also, I implemented hover effects on
navigation bar to enhance the user experience and accessibility.

Challenges & Lessons Learned

Ensuring responsiveness

One of the biggest challenges I faced when building the home page was making sure it was displayed
properly and responsive, to make sure that elements like the navigation bar, images, and footer adapted
well to different screen sizes took me some extra effort.

Lesson learnt

I learnt the important of responsive design techniques, such as media queries in CSS, and testing the
design on different devices using Inspect Element

CSS Styling Consistency

Ensuring that the CSS was consistency across all pages was difficult, especially with buttons,
links, and the navigation bar.

Lesson learnt

Using a global CSS stylesheet made it easy for everyone to know what global style, fonts and colours to utilise

Compliance

The home page complies with the Janet regulations by following accessibility. I made sure the page met
the accessibility guidelines by utiling a navigation bar and hover effects. This approach made sure that the home
page was accessible for all users.

Content Page

Purpose: The Wisdm Content Page presents an overview of Wisdm's educational activities, with a focus on significant projects that promote quality education. It describes various major programs, including digital literacy for adults in developing countries, STEM education for women, teacher training initiatives, and inclusive education for pupils with special needs. The page is intended to educate visitors about these programs and illustrate their impact in various areas. In addition, the page has an intuitive structure with graphics, internal links for easy navigation, and a sidebar to help readers rapidly access key areas. This structure provides an entertaining and instructive user experience while also supporting Wisdm's education program.

Content Page's HTML / CSS Structure and Key Features

The content page is structured in a straightforward, accessible manner, with semantic HTML elements such as header, nav, main, and footer ensuring a logical flow for both people and search engines. The usage of section elements organises the content into separate categories, such as "Introduction," "Digital Literacy," and "Teacher Training Initiative," making it easier to navigate. Anchor items in the navigation menu link to various portions of the page, enhancing the user experience by providing direct access to crucial material. Headings, such as h1, h2, h3, and h5, provide a clear hierarchy, allowing visitors to easily comprehend the structure of the material.

CSS like flexbox are used to build a responsive and visually appealing layout, ensuring that the page adapts to different screen sizes. Font family, colour, and margin are examples of styling properties that are used to improve readability and make text visually appealing. Images are labelled with alt text for accessibility, ensuring that visually challenged people can access the information communicated by images. The sidebar with internal links lets visitors to swiftly navigate between parts, improving navigation without disrupting the page's general flow. Overall, these strategies help to create an organised, user-friendly, and accessible information page.

Content Page's Accessibility

The table page is designed with accessibility in mind to make sure that all users, including those with disabilities, can access and navigate the content effectively.. The table layout is structured using HTML elements like table, thead, and tbody. The use of tooltips through the title attribute allows for more information to be provided without overwhelming the pag econtent.

Challenges & Lessons Learned

One of the most difficult aspects of implementing the content page was ensuring that the layout was clean and user-friendly, especially with the sidebar for internal navigation. Initially, the sidebar overlapped with the main content on certain screen sizes, resulting in a cluttered appearance. To address this, I utilised CSS flexbox to build a more flexible and organised structure, ensuring that the sidebar and content section were properly spaced across all screen widths. Additionally, aligning graphics within text and ensuring they did not disrupt the flow of the information was difficult at first. This was addressed utilising CSS features such as 'float', 'clear', and'margin' to ensure proper placement and spacing. Another problem was keeping the design similar across multiple browsers.

These problems taught me the value of testing and changing the layout at various phases of development. I also learnt more about CSS for complicated layouts, as well as how tiny stylistic changes may significantly improve the overall user experience. This experience taught me to always prioritise visual flow and text clarity, ensuring that the design adapts effectively to different screen sizes and is functional across several browsers.

Compliance

The content page's compliance with online standards and laws was a top priority. The page was developed with accessibility in mind, adhering to the Web Content Accessibility Guidelines. All photos have alternative text (alt text), allowing people with visual impairments to interpret the material. The usage of semantic HTML tags like headers, paragraphs, and sections guarantees that the material is user-friendly and search engine optimised. In terms of Janet regulations governing web page publication, the page complies with standards by utilising correct, well-structured HTML and CSS to improve reading and navigation. The page design is responsive, which means that the content appears nicely on multiple devices and screen sizes.
